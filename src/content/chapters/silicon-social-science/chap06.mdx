## 6.1 中心收缩比的定义

<Definition number="6.1" title="中心收缩比">

对一个 ordinal 量表 $K$ 档，把回答编码为整数 $1, \ldots, K$。设 silicon 与真实回答的标准差分别为 $\sigma_{\mathrm{silicon}}$ 和 $\sigma_{\mathrm{real}}$。中心收缩比为

$$
\mathrm{Shrink} = \frac{\sigma_{\mathrm{silicon}}}{\sigma_{\mathrm{real}}}.
$$

取值范围 $(0, \infty)$。Shrink $< 1$ 表示 silicon 分布比真实更"窄"，质量向众数集中；Shrink $> 1$ 表示 silicon 比真实更分散；Shrink $= 1$ 表示方差结构对齐。

</Definition>

其中 $\sigma$ 是数值化 ordinal 量表上的标准差。该量度只用样本一阶矩外的二阶矩信息，对边际众数无关，专测"分布是否被压向中央"。

### 6.1.1 polviews 上的中心收缩比

```r
sd_real <- sd(df$polviews_ord, na.rm = TRUE)
sd_sil  <- sd(df$polviews_si_ord, na.rm = TRUE)
shrink <- sd_sil / sd_real
round(c(sd_real = sd_real, sd_silicon = sd_sil, shrink = shrink), 3)
# sd_real = 1.414, sd_silicon = 0.859, shrink = 0.607
```

silicon 的 polviews 标准差是 0.86，真实是 1.41，收缩比 **0.61**。silicon 把 polviews 的离散度压到了真实分布的 61%——分布变窄、质量向 Moderate 一档集中。

| 档 | 真实 % | silicon % | 差 |
|:---|-------:|----------:|---:|
| Extremely Liberal      |  3 |  0 | $-3$  |
| Liberal                | 14 |  5 | $-9$  |
| Slightly Liberal       | 11 |  7 | $-4$  |
| **Moderate**           | **42** | **71** | **+29** |
| Slightly Conservative  | 12 |  7 | $-5$  |
| Conservative           | 14 | 10 | $-4$  |
| Extremely Conservative |  4 |  0 | $-4$  |

silicon 把所有非 Moderate 档都缩水，Moderate 一档 +29 个百分点。极端档（Extremely Liberal + Extremely Conservative）silicon 占比 0%，真实 7%。这意味着**政治极化**这一研究主题如果用 silicon sample，结论会完全失真——silicon 根本不会给出极端立场的受访者。

## 6.3 中心收缩的形成机制

中心收缩在 LLM ordinal task 上是已知系统性现象。原因有三层。

**第一**：LLM 的训练数据里"自我标识为极端立场"的比例比 GSS 真实分布低。互联网文本里大部分用户在写带政治立场的文章时倾向"理性中立"叙事，"我是 Extremely Liberal"这种直接极端自我标签出现频率低。LLM 在 silicon sampling 时把这种"低先验"作用到每个 persona。

**第二**：LLM 的 RLHF（人类反馈强化学习）训练显式鼓励"中立、平衡、避免极端"表述。这一训练目标在 silicon sampling 时跨任务转移——LLM 倾向于给出"温和的"答案，即使该答案不符合 demographic 信号。

**第三**：silicon sampling 的 prompt 给的人口学画像没有"政治意识形态强度"的直接线索。LLM 必须从年龄 / 种族 / 地区 / 学历推断，这些 demographic 与"极端档"的相关性统计上确实弱（GSS 真实数据上 demographics 解释 Extremely 档的 R² 不到 0.05），LLM 更倾向于把 ambiguous case 压向众数 Moderate。

## 6.4 Distributional Repair：能修吗

发现 silicon 有中心收缩后，自然问能不能事后修复。一种思路是 *distributional repair*——把 silicon 的边际分布重新采样到与真实匹配的形状，但保留 silicon 给个体打的相对顺序。具体做法是把 silicon 的 100 个 polviews 排序，按真实分布的分位数重新赋档。

```r
# 把 silicon polviews 重新分位赋档
real_quantiles <- quantile(df$polviews_ord, probs = seq(0, 1, length.out = 8),
                            na.rm = TRUE)
df$polviews_si_repaired <- cut(rank(df$polviews_si_ord, ties.method = "average"),
                                breaks = quantile(seq_along(df$polviews_si_ord),
                                                  probs = seq(0, 1, length.out = 8)),
                                include.lowest = TRUE, labels = 1:7)
```

distributional repair 能把 silicon 的边际分布对齐到真实，但对个体级准确率没改善——repair 后的个体匹配率不变，只是把"全压 Moderate"重新分散开。这种 repair 在"群体级建模"任务上有用（如训练统计模型时需要分布匹配），在"个体级研究"上没用。

repair 的根本问题在于 silicon 把 demographics 与 attitude 的关联学得太单一。重新分位赋档只是改变 marginal，没有改变 silicon 对每个 persona 给出的 attitude 排序。如果排序本身就有系统偏差（如把 White Female 排在更民主一端），repair 后这种偏差仍然存在。

<Pitfall title="用 distributional repair 当 silicon 的'修复方案'">

看到中心收缩后，自然想"修一下分布形状"。distributional repair 能让 silicon 与真实的 TVD 接近零，但个体准确率不变，subgroup bias 不变。

**为什么人们会踩坑**：marginal 对齐看起来是"修好了"，TVD 数字会显著下降。

**失效机制**：repair 只动 marginal 分布，不动 silicon 内部的"demographics → attitude" mapping。所有第 5 章的 subgroup bias 在 repair 之后照旧。

**诊断方法 / 稳健替代**：不要用 repair 当作"silicon 修复完毕"的标志。如果要做群体级建模可以 repair 后再用 silicon 做 ABC（approximate Bayesian computation）这种不靠个体准确率的方法；做个体级或子群研究坚决不要 repair。

</Pitfall>

## 6.5 中心收缩的"温度"假说

一个常见假说是 silicon 中心收缩是 *temperature 参数* 过低导致的。LLM API 调用时把 temperature 调高（如 1.5）可能让 silicon 在分布尾部更活跃。这一假说在第 7 章 prompt 设计章详细检验，结论是：温度只能小幅改善中心收缩，根本机制是 RLHF 加上训练分布中立偏好，不是采样温度。
