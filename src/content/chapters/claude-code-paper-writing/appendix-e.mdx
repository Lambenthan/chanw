使用 Claude Code 辅助论文写作时最常见的问题与对应的处理方法。

---

## 1. Claude Code 修改了术语

**现象**:润色过程中专业术语被替换为同义词(第 1 章详述了此类事件)。

**原因**:它不知道哪些词在本论文中不可改动。

**处理方法**:在 CLAUDE.md 中维护术语锁定表(见第 2 章)。每次修改任务的指令中也再次列出关键术语。参考附录 D 的模板。

---

## 2. 修改了未指定的位置

**现象**:让它改第三段的措辞,它顺手修改第四段的一句话,还删除了一个引用编号。

**原因**:指令未明确限定修改范围。

**处理方法**:指令中明确"只修改第 X 段,其他段落不要动"。在 CLAUDE.md 中加一条"修改正文时保留所有引用编号"。改完后用 diff 对比改前改后,确认只改了指定位置。

---

## 3. 会话中途被 rate limit 打断

**现象**:运行中,Claude Code 提示用量到上限,整个会话停止。

**原因**:Pro 版用量有限,长任务容易超限。

**处理方法**:升级到 Max 5x 可减少被打断的频率。更重要的是在指令中要求维护检查点(详见第 10 章)。被打断后用 `claude --continue` 续接会话,它能读取检查点文件并从断点继续。

---

## 4. 编造文献观点或数据

**现象**:让它写文献综述,它写了一句"某某研究发现……",查阅原文后发现该论文未提及此事。

**原因**:它根据标题与已知信息推测文献内容,推测未必正确。

**处理方法**:在 CLAUDE.md 中加一条"不确定的内容标注 [需核实]"。所有涉及文献评述的内容,均回到原文核实。参考第 14 章第 2 节。

---

## 5. DOCX 文件被覆盖,原文丢失

**现象**:让它修改 Word 文件,它直接覆盖原文件,导师批注与之前的修改全部丢失。

**原因**:Claude Code 默认直接写回原文件。

**处理方法**:在 CLAUDE.md 中加一条"修改任何 .docx 文件前先创建备份"。或配置一个自动备份的 Hook(第 11 章讨论)。更稳妥的做法是尽早迁移到 LaTeX 工作流。

---

## 6. Draw.io 图的 MCP session 丢失

**现象**:花了半小时绘制好一张图,切换到下一个话题后回来,图不见了。

**原因**:Draw.io MCP 的 session 状态不跨 turn 保存。

**处理方法**:每次图修改到满意状态,立即让 Claude Code 把 XML 保存到本地文件。用文件作为唯一真相来源。下次修改时让它从文件读取。参考第 6 章的详细说明。

---

## 7. 长句未被拆开

**现象**:让它改写一段话,改完仍有不少超过 50 字的长句。

**原因**:指令未明确句子长度要求。

**处理方法**:指令中加一条"每句不超过 25 字"或"找出所有超过 50 字的句子,逐句改短"。若经常做此类任务,把句子长度要求写入 CLAUDE.md 的写作规范。

---

## 8. 改完结果与导师要求不符

**现象**:导师说"结论太弱",让 Claude Code 改,改完导师看了说"不是这个意思"。

**原因**:导师的意见是概括性的,未翻译成具体修改要求即直接交给 AI。

**处理方法**:收到导师意见后,先想清楚每条意见的具体含义与修改方向,把自身理解写成具体指令再交给 Claude Code。参考第 14 章第 1 节的例子与附录 A 的"导师反馈执行"模板。

---

## 9. 全文术语不一致未被发现

**现象**:第二章使用"被解释变量",第四章使用"因变量",自行通读未察觉。

**原因**:作者多次通读自己的论文后会产生熟悉感,看到不一致也不觉得异常。

**处理方法**:用并行 Agent 跑一次全文术语一致性检查(第 10 章 §10.4),让它列出所有不一致位置。

---

## 10. Word 表格修改后错位

**现象**:让 Claude Code 给 Word 表格加一行,完成后列对不齐、单元格边框消失。

**原因**:DOCX 的表格在 XML 中是结构化嵌套的,修改时容易破坏底层结构。

**处理方法**:复杂格式文档让它只改纯文字部分,不动表格。若必须修改表格,每次只改一处即验证。若频繁遇到此类问题,考虑迁移到 LaTeX(第 8 章详述)。
