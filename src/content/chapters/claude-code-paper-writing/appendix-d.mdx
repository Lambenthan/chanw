以下为面向科研论文写作的 CLAUDE.md 模板。可直接复制到项目文件夹根目录,再根据自身情况修改方括号中的内容。

```markdown
# CLAUDE.md

## 项目基本信息
- 论文题目:[论文题目]
- 研究方向:[一句话描述]
- 当前阶段:[初稿 / 修改中 / 定稿 / 答辩准备]
- 论文格式:[LaTeX / Word]
- 提交要求:[学校的格式规范,例如字体、行距、页边距]

## 文件结构
- chapters/ — 各章正文
- tables/ — 所有表格文件
- figures/ — 所有图表文件
- references/ — BibTeX 文件与参考文献 PDF
- appendix/ — 附录文件
- backups/ — 备份目录(gitignore 排除)

## 术语锁定表
以下术语不得修改或替换:
- [术语 1]
- [术语 2]
- [术语 3]
若你认为某个术语需要修改,先问我,不要自行修改。

## 写作规范
- 使用学术中文,不要口语化表达
- 每段只讲一件事
- 句子不超过 25 字
- 不使用「由于……因此……」的长句式
- 不使用「值得注意的是」「具有重要意义」等空洞修饰
- 段与段之间不需要刻意的过渡句,直接进入内容
- 图表标题格式:图 X-Y:标题内容
- 引用格式:GB/T 7714

## 修改规则
- 修改正文时保留所有引用编号
- 不要删除任何引用
- 修改之前先告诉我方案,确认后再执行
- 改完给我看修改前后的对比
- 修改任何 .docx 文件前,先创建备份到 backups/

## 导师偏好
- [导师对写作风格的具体要求,例如「一段一个主题」]
- [导师的用词禁忌,例如「不使用 实证分析 / 显然 / 易得」]
- [导师对格式的要求,例如「每章另起一页」]

## 常见错误防线
- 不要把「被解释变量」改成「因变量」
- 不要删掉引用编号
- 不要编造文献观点,不确定的标注 [需核实]
- [其他踩过的问题,每出现一次追加一条]

## 涉及数据或文献时的来源标注
撰写涉及数据、文献或实证结果的段落时,
若某句话的信息不来自我提供的材料,在句末标注 [需核实]。
```

---

## 使用建议

**此模板是起点,而非最终版**。使用过程中发现需补充的规则,随时追加。Claude Code 每次启动会话时读取此文件,追加的规则下次会话即生效。

**术语锁定表要优先填好**。论文中的核心术语必须列入。判断某个词算不算核心术语,可以看它被 AI 替换成同义词后会不会导致全文不一致,会的就列进来。

**"常见错误防线"一栏**,每次 Claude Code 出现不希望第二次看到的错误,即追加一条。半年下来该栏会较长,但每条均有用。

**"导师偏好"一栏**,把导师开会时提过的具体要求记入。例如"摘要不放具体数字"、"标题不用问句"、"表格不跨页"。Claude Code 了解这些要求后,改稿时会自动遵守。

<GhAlert type="tip">

**CLAUDE.md 可加入 Git 管理**

</GhAlert>

>
> 若论文文件夹使用 git 管理(或计划使用 LaTeX 加 git),CLAUDE.md 可提交至 git。这样换电脑或重装系统时,CLAUDE.md 随项目同步,不会丢失。
