## 一、找到并确认一个连本地 Stata 的 MCP 工具

会话的第一句问的是有没有适合 Claude Code 用的 Stata MCP 工具。查下来，当前环境里其实已经接好了一个，系统提示里能看到 `mcp__stata-mcp__*` 这一系列工具。它来自 PyPI 上的 **`stata-mcp`** 包，作者是 Sepine Tam，通过 `uvx stata-mcp` 启动，配置写在 `~/.claude.json` 的 `mcpServers` 字段里，安装命令一行就够：`claude mcp add stata-mcp -- uvx stata-mcp`。

先跑一个最轻的命令确认进程能起。让它调 `help describe`，返回了 `describe` 命令的完整文档，带 PDF 手册链接和 Stata/MP/SE 的提示，说明本地 Stata 进程能正常拉起，返回也是当场生成、没有走缓存。装的版本从 PDF 手册路径推断是 **Stata 18**。这个 MCP 对外提供 **五个工具**：`stata_do` 执行 do 文件并返回日志，是主力；`get_data_info` 读 dta、csv、xlsx 的结构与预览；`read_log` 解析落盘的 log 与 smcl；`help` 查命令文档；`ado_package_install` 装 SSC 或 GitHub 第三方包。

接着要弄清一个容易误解的地方：它连的是本地的 Stata，不是云端。macOS 的自动发现逻辑会先扫 `/usr/local/bin`，再扫 `/Applications/Stata*`，按版本号取最高的，本机命中的可执行文件是 `/Applications/Stata/StataMP.app/Contents/MacOS/stata-mp`。要紧的是它和你手动打开的那个 StataMP 图形界面是**两个互相独立的进程**。GUI 有自己的内存、数据集和 Results 窗口；MCP 每次调 `stata_do` 拉起的是 batch 模式的命令行进程，跑完即退，两边不共享内存、不共享数据、不共享日志。你在 GUI 里 `use` 的数据，MCP 那边看不见；MCP 跑的回归，GUI 的 Results 窗口里也不出现。所以 MCP 跑出来的结果默认只能通过 Claude Code 回传，落地形式是每次写到项目 `.statamcp/stata-mcp-log/` 下的 `.log` 文本与 `.smcl` 带格式日志，在 GUI 里 `view` 那个 smcl 也能看到同样的字。

这个工具自带一道沙箱：do 文件必须放在它允许的目录里。会话里第一次把探测用的 do 文件写到 `/tmp` 就被拒了，报 `Access denied: outside allowed directories`，把文件移进项目下的 `.statamcp/stata-mcp-dofile/` 才跑通。后面所有 do 文件都落在这个目录。

## 二、把三套原始数据导入并清洗成面板

研究计划确认在项目里之后，下一句指令是看项目里的数据能不能支持完成全部研究，并要求调 Stata MCP 把每一步都做出来。Claude Code 先进计划模式盘点数据，再分阶段执行。原始数据散在七个模块二十多个文件里，dta、xlsx、csv 混着，字段命名也各不相同，从这堆原料到一份能跑回归的面板，要按主键合并、去重、处理缺失、构造变量，最后跑描述统计核样本面貌。

![从三套原始数据到分析面板的清洗流程：华证 ESG、CSMAR 财务、管理者短视三套数据各自导入并构造变量，按 stkcd 与年度合并、缩尾、筛样本，落成 26874 obs 的面板，每一步规则由研究者定、执行交给 Claude Code 写 do 文件](/images/essays/claude-code-empirical-workflow-04-cleaning@2x.png)

第一步是装包建目录。do0 用 `ado_package_install` 装一批 SSC 包，`reghdfe`、`ftools` 一次到位，`estout` 和 `winsor2` 第一次失败，单独重试才装上，随后 `outreg2`、`ivreg2`、`ranktest` 也补齐。装包失败再重试这样的反复在实际操作里常见，靠日志看清哪个包没装上才好补。

每导一套数据之前，先 probe 一遍字段，再写正式的导入脚本。这是清洗里很要紧的习惯，因为原始文件的列名、脏数据、单位都得先看清。**华证 ESG** 那份文件名里带空格，是 `华证指数esg 评级评分（年度） 2009-2024.xlsx`，导进来 **52,453 obs**，有 E、S、G 三个分项，综合赋值已是数值化等级，综合得分是字符串需要 `destring`，时间字段里还混进了字面值"时间"这样的脏数据要清掉。清完落盘 **52,452 obs**，2009 到 2024 全覆盖，esg_score 均值 0.73。**CSMAR 财务**这一组 probe 时发现一个缺口：利润表 FS_Comins 只导出了营业收入，**没有净利润**，ROA 因此构造不出来，改用经营现金流比总资产的 Cash 作盈利能力代理，并在文档里写明这处数据局限。这套面板 **68,848 obs**，Size、Top1、Soe 分布正常，Lev、Cash、Growth 有极端值留待缩尾。**管理者短视**用的是 MDA 版，**57,838 obs**，主指标取去表后关键词词频的 `Deratio2`，对应胡楠 2021 的主用口径，乘 100 得到 myopia100。

把三套按 `stkcd` 和会计年度合并成主面板，是清洗里容易出隐性错误的一步，样本量莫名其妙少几百行往往就出在合并和缺失处理上，靠日志逐步回查最稳妥。合并后对 Lev、Cash、Growth、Top1、KZ、myopia、Size 这些连续变量按 1% 和 99% 分位缩尾，最终落成 **26,874 obs、3,608 家公司、2009 到 2023** 的面板，落在文献预期区间。这里每一处口径都是研究判断：用哪个键合并、缺失是删是补、异常值缩不缩尾，Claude Code 写得了脚本，定不了规则。喂材料时最关键的一份是变量字典，写清每个变量在哪个文件、什么类型、什么单位，再给出目标面板的形状，脚本才贴合。

## 三、描述统计与主回归

面板立住后出描述统计。do5 用 estout 出表 1 描述统计和表 2 相关性，均值、标准差、最大最小、样本量列成一张表。一个值得记下的信号是单变量层面 **PC 与 ESG 的 Pearson 相关已显著为正，ρ=0.088、p < 0.001**，主回归预期能成。Spearman 那行因为 estpost 不支持报了错，不影响主交付，照实跳过。

![从研究假设到主回归的工作流：研究者先定被解释变量、核心解释变量、控制变量、固定效应与聚类层级，再让 Claude Code 写 do 文件、由 stata_do 执行、把日志回读解读，Stata 在旁交叉复核](/images/essays/claude-code-empirical-workflow-04-regression@2x.png)

主回归用 reghdfe 跑双向固定效应、按公司聚类标准误。第一次跑报 reghdfe 缺 `require` 依赖，装上再跑。模型设定里每一项都来自研究判断：被解释变量是 esg_score，核心解释变量是耐心资本 PC，控制变量是 Size、Lev、Cash、Growth、Top1、Soe、Age、KZ 这一组，固定效应控制到公司和年度，标准误在公司层聚类。这些定下来，代码 Claude Code 写得又快又准。

跑完有个要明示给研究者的发现：主回归口径的选择直接改变结论。**A2 主口径，即股权与债权两侧标准化后加和，不显著，β=0.0004、p=0.235**；而 **A1 经典版显著，β=0.0018、p < 0.001**；C 熵值法 β=-0.0024 也不显著。PC 内部分解进一步显示债权侧主导。同一份数据、换一种耐心资本的测算方式，系数的显著性就翻面，口径选择因此留给研究者。Claude Code 把三种口径都跑出来如实摆上，选哪个进主表是人的事。

结果解读是 AI 痕迹的高发处。让它写系数解读，常写出形容词堆叠、结论拔高的腔调。喂材料时把这层管住：给本领域论文的实证段落做范本，要求按平实语气写，系数多大、在什么水平显著、方向跟假设是否一致，照实陈述，变量名沿用研究里既定的说法。主回归同样走交叉复核，Claude Code 跑出来的结果，自己再用 Stata 手动跑一遍同样的设定，系数、标准误、样本量逐一对齐，两端一致才算立得住。

## 四、内生性、机制、异质性与稳健性

主回归立住后，第二阶段把内生性处理、机制检验、异质性、稳健性一组跑齐。这些检验彼此独立，正好批量做。

![第二阶段四组检验的结构：IV 与 PSM 处理内生性、KZ 与 Myopia 走机制、四维分组做异质性、测度替换与滞后期做稳健性，各组独立批量跑，研究者判断哪组对识别策略关键](/images/essays/claude-code-empirical-workflow-04-robustness@2x.png)

内生性这组用工具变量和倾向得分匹配。IV 跑出来 **同行业均值作工具变量 IV1 显著，β=0.0052、Kleibergen-Paap Wald F=210.3**，PC 滞后作工具变量 IV3 也显著，β=0.0021，Hansen J 的 p=0.057。PSM 的 **ATE=0.0032、t=3.98**，组间比较显著；进了固定效应后 β=0.0004 不显著，说明效应主要在公司横截面。Placebo 把处理变量随机置换 500 次，实际系数大于全部置换系数，p=0。这一组跑的过程也不是一帆风顺：`bysort` 改了排序导致滞后算子 `L.` 出错，先 sort 再算才对；`xtivreg2` 没装，补装；嵌套 `preserve` 报 r(621)，拆成 do7b 收尾。

机制检验沿 KZ 融资约束和管理者短视两条通道走，结果出来两处跟预期不符：PC 与 Myopia 是**正向**关系，跟预期的负向相反；Sobel 检验里 Myopia 通道 z=-3.09、p=0.002 显著，但间接效应几乎为零。这种与假设不符的结果照实交付，让研究者判断是机制设定问题还是理论本身需要修正，工具不替人圆结论。异质性按规模、杠杆、产权、上市时长四维分组，核心发现是**非国有企业的 PC 系数 β=0.0009 显著**，与假设一致，其余三维分组差异不显著。异质性比稳健性更要判断：分组之前得先有一条说得通的机制假设，先有机制再分组验证才有意义，为找显著结果到处试分组就成了数据挖掘，而机制从哪来落回文献和理论。

稳健性矩阵替换 PC 测度、换 ESG 评级的离散口径、加滞后期。一个值得记的结果是 **PC 滞后 1 期 β=0.0010、滞后 2 期 β=0.0008 都显著**，说明耐心资本对 ESG 是一到两年的滞后效应。写稳健性段落有个朴素标准：每个结论后面都跟一句"这说明……"，替换测度后仍显著就写清结论不依赖特定口径，滞后期显著就写清效应有时滞。让 Claude Code 起草这些段落时把要求交代下去，再自己核每一句"这说明"是否站得住。

## 五、把回归结果导成三线表 Word

研究计划里点名要 Word 文档、每张表用三线表。这一步遇到一个工具限制，值得完整记下。第一版用 Stata 的 `putdocx textfile` 把 esttab 导出的 RTF 表嵌进 docx，生成的文档里表格全是乱码。拆开 docx 的 XML 一看，`<w:tbl>` 个数是 **0**，183 段 RTF 控制码以裸文本形式塞了进去。`putdocx textfile` 没把 RTF 转成 Word 表格，而是当纯文本嵌入，控制码裸露成了用户看到的"乱码"。这是 putdocx 的已知限制。

换路线。先写一个 do 文件用 esttab 把五张表导成 CSV，中文在 CSV 里完全正常，绕开了 RTF 的中文转义问题。再用 Node 的 `docx` 库写脚本读 CSV，版本是 9.6.1，**原生构造 `<w:tbl>` 三线表**：表格默认无边框，表头单元格顶部一条 12pt 黑线作顶线、表头底部一条 8pt 黑线作中线、数据末行一条 12pt 黑线作底线，全程无竖线，中文用 SimSun 与 SimHei、西文数字用 Times New Roman。生成的文档过了 docx skill 自带的 validate 校验，第二阶段全部跑完后合到一份 28.9 KB 的 Word，含 **12 张原生三线表**、0 段 RTF 残留、校验全过。

这里有一条通用经验：让 Claude Code 产出格式化交付物时，中间格式选错会整段返工。CSV 这种纯文本中转比让工具直接吐富文本稳妥，最终格式由能精确控制结构的库来生成。具体用哪条路线由研究者拍板，docx skill 的推荐和 putdocx 的限制都摆出来，选 docx-js 重建是看清根因后的决定。

## 六、整条流水线的分工与交叉校验

把这几段连起来看，从找工具、接 Stata MCP，到导入清洗、描述统计、主回归、稳健性、出表，研究者与 Claude Code 的分工是清晰的。

![实证研究后半程的三栏分工总览表：找配 MCP 工具、数据导入清洗、描述统计、主回归、内生性与稳健性、结果出表各环节，左列是研究者拍板的判断如口径选择与机制假设，右列是 Claude Code 与 Stata 接手的执行如写 do 文件、跑回归、导三线表](/images/essays/claude-code-empirical-workflow-04-overview@2x.png)

越靠近定方向的环节，比如耐心资本用哪个测算口径、控制变量放哪些、识别策略怎么设、机制假设站不站得住，Claude Code 能帮的越少，这些是研究的内核，判断权在研究者手里。越靠近做执行的环节，比如装包、写 do 文件、跑回归、批量出稳健性、导三线表，它接手得越彻底。一篇实证论文的价值落在前者，效率的提升来自后者。

整个系列反复回到同一个收尾动作。从清洗到主回归再到稳健性，每个关键结果都让 Claude Code 调 Stata MCP 跑一遍、留下完整 do 文件，主回归再由研究者用 Stata 手动复跑核对，两端结果相互对齐才落进论文。A2 与 A1 口径系数翻面、PSM 显著而固定效应内不显著、机制方向与预期相反，这些都是 Claude Code 如实回传、由研究者判断的结果，工具不替人下结论。

四篇到这里收尾。工具摆对位置、流程看全、项目管好，再把文献调研、研究计划、数据导入清洗、回归与稳健性逐段走通，一篇实证论文从开题到出表就有了完整的操作路径。Claude Code 把执行接成连续的流水线，研究的判断始终留在研究者这边，这是这个系列从头到尾想说清楚的一件事。
